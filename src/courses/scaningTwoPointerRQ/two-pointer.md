# 雙指針技巧

假設現在有兩個排序過的陣列，各自的長度為 $n, m$，
如果你要得到一個兩者合成的排序陣列，要怎麼做呢？

直接講兩個陣列合併後排序，需要 $\mathcal{O}((n + m) \log (n + m))$ 的時間，
但是既然兩個陣列已經各自排序過了，我們可以善用這個性質。

假設兩個陣列是有序且不重複的，
那麼最小的數值肯定是兩個陣列頭部元素之一，不用去考慮第 2, 3, ... 個元素，因為有序的性質，
它們肯定比第一個數字大，所以每次只要比較目前兩個陣列中還沒被選到的數字，就可以完成合併。

以 $a = [1, 3, 5], b = [2, 4, 6]$ 為例，
比較 $a_0$ 和 $b_0$，因為 $2 > 1$，所以先取出 $1$，
然後比較 $a_1$ 和 $b_0$，因為 $3 > 2$，所以取出 $2$，
然後比較 $a_1$ 和 $b_1$，因為 $3 < 4$，所以取出 $3$，
然後比較 $a_2$ 和 $b_1$，因為 $5 > 4$，所以取出 $4$，
然後比較 $a_2$ 和 $b_2$，因為 $5 < 6$，所以取出 $5$，最後只剩下 $b_2$，所以取出 $6$，完成合併。

這樣可以在 $\mathcal{O}(n + m)$ 的時間內完成合併，雖然大部分情況下 $\mathcal{O}((n + m) \log (n + m))$ 的時間是可以接受的，
不過這裡的目的是帶出雙指針的概念。

在剛才合併陣列的時候，我們使用兩個變數各自紀錄兩個陣列比較到的索引，
這個就是所謂的指針。

因為陣列是排序過的，所以有所謂的 **單調性**，跟二分搜尋法時類似，
當我們在推進指針時，狀態的變化是單向的，這意味著指針只會一直往前走，絕對不需要回頭。
如果沒有單調性的保證，我們在比對資料時可能需要依賴雙重迴圈（即 $\mathcal{O}(n \times m)$），當其中一個陣列的索引前進時，另一個陣列的指針就被迫要退回起點重新掃描。

但正因為序列是遞增的，當我們比較 $a_i$ 與 $b_j$ 並決定把較小的 $a_i$ 放進結果後，我們很清楚已經被取走的 $a_i$ 絕對比剩下的任何數字都小，因此 $b$ 陣列的指針 $j$ 繼續停留在原地等待下一次比較即可，完全不用往回退。

這種指針「只進不退」的特性，保證了兩個陣列中的每個元素最多只會被評估與讀取一次，這正是雙指針技巧能將時間複雜度降到線性時間 $\mathcal{O}(n + m)$ 的核心概念。

像這類兩個變數從同側起點出發、朝同一個方向移動的操作，我們通常稱為「同向指針」。而根據題目單調性的不同設計（例如在有序陣列中尋找特定總和的兩個數），雙指針還有另一種常見的變體：將變數分別放在陣列頭尾、彼此向中間靠攏的「相向指針」。
總結來說，掌握雙指針技巧的核心，就是學會觀察資料中隱含的單調性，並藉由指針單向移動的策略，來剔除掉大量不必要的重複計算。

## 同向指針

~~~admonish note title="問題描述"
給定一個長度為 $n$ 的整數陣列 $a$，
請求出最長不包含重複元素的子陣列長度。
#### 技術規格

- $1 \leq n \leq 2 \times 10^5$
- $-10^9 \leq a[i] \leq 10^9$
~~~

我們先用一個實際的例子，並用最暴力的雙層迴圈法來探討這個問題。

若 $a = [1, 2, 3, 1, 4]$，且 $L = 0, R = 2$，此時子陣列 $a[L..R]$ 為 $[1, 2, 3]$，其中沒有重複元素，長度為 $3$。

下一個元素 $a_3$ 為 $1$，掃描到 $L = 0, R = 3$，此時子陣列 $a[L..R]$ 為 $[1, 2, 3, 1]$，其中有重複元素，應該停止當前輪次的遍歷，

通常而言，我們會更新 $L = 1$，且將 $R$ 重置為 $L$，即 $R = 1$，然後繼續遍歷。

但是這裡有一個有趣的性質，$a[1..1]$ 是 $a[0..2]$ 的子陣列，既然 $a[0..2]$ 沒有重複的元素，則 $a[1..1]$ 必然也沒有重複的元素，
這就是這道題目可以利用的性質，當一個區間滿足條件時，其子區間也會滿足，所以我們不需要將 $R$ 重置為 $L$，而是繼續嘗試能否擴展即可。

因為左右兩指針都不會回退，所以時間複雜度是比起雙層迴圈 $\mathcal{O}(n^2)$ 更加優秀的 $\mathcal{O}(n)$。

```kotlin
import kotlin.math.max
fun longestSubarrayWithoutRepeating(a: IntArray): Int {
    var right = 0
    var res = 0
    var st = HashSet<Int>()
    for (left in a.indices) {
        while (right < a.size && !st.contains(a[right])) {
            st.add(a[right])
            right++
        }
        // 跳出迴圈時，right 是不合法的
        res = max(res, right - left)
        st.remove(a[left])
    }
    return res
}
```

使用一個 `HashSet` 來記錄當前窗口中的元素，並在遍歷過程中動態更新窗口大小。

上面的實作是根據我們的教學來呈現的，但是需要判斷 `right` 是否超出陣列範圍，有點冗餘，我個人更喜歡下面的寫法

```kotlin
import kotlin.math.max

fun longestSubarrayWithoutRepeating(a: IntArray): Int {
    var left = 0
    var res = 0
    var st = HashSet<Int>()
    
    for (right in a.indices) {
        // 當發現重複元素時，左指針不斷向右移，直到該重複元素被移出集合
        while (st.contains(a[right])) {
            st.remove(a[left])
            left++
        }
        
        // 記錄當下有效的最大視窗長度
        res = max(res, right - left + 1)
        
        // 將當前的元素加入集合中
        st.add(a[right])
    }
    
    return res
}
```

## 相向指針

既然我們已經認識了同向指針的使用環境，接著就要來看何謂相向指針。

~~~admonish note title="問題描述"
給定一個長度為 $n$ 的整數陣列 $a$，
陣列已經經過排序，請問能否找到一對數組 $(i, j)$，使得 $a[i] + a[j] = target$，

其中 $target$ 是一個給定的整數。
#### 技術規格

- $1 \leq n \leq 2 \times 10^5$
- $-10^9 \leq a[i], \text{target} \leq 10^9$
~~~


我們一樣用雙層迴圈的思考模式，當前 $i$ 指向陣列的起始位置，$j$ 則從 $i + 1$ 開始向右移動，
因為陣列是排序過的，如果對目前的 $a[i]$ 而言，$a[i] + a[j] < target$，則 $j$ 可以向右移動，尋找更大的數字來湊出目標值;
若 $a[i] + a[j] > target$，則該輪應該由此結束，$j$ 繼續往右數值只會越來越大。

這裡有一個很有趣的地方，當我們 $i$ 進入到下一輪時，因為 $a[i]$ 變大了，所以 $j$ 最後會終止的地方只會往內縮，

也就是說，與其把 $j$ 從 $i + 1$ 開始往右移動，不如一開始就放在最右邊，視情況來往左移動。

一開始兩個指針在陣列的左右兩側，當 $a[i] + a[j] < target$ 時，$i$ 向右移動，因為這樣可以讓總和變大，
當 $a[i] + a[j] > target$ 時，$j$ 向左移動，因為這樣可以讓總和變小，直到找到目標，或者 $i$ 與 $j$ 相遇。

這個就是相向指針的應用，透過利用陣列的有序性質，將一個原先需要 $\mathcal{O}(n^2)$ 的時間複雜度，降低到 $\mathcal{O}(n)$。

## 結論

無論是同向指針或者是相向指針，雙指針的應用就是當我們進行暴力列舉時，發現了一些有用的性質，
透過性質的利用可以大幅減少無效的嘗試，
